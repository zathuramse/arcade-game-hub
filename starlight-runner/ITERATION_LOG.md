# Iteration Log

## v1 - 平台核心

- 建立新專案 `starlight-runner`。
- 完成橫向卷軸 Canvas 舞台、角色、重力、左右移動、衝刺與跳躍。
- 實作跳躍緩衝與土狼時間，改善平台跳躍手感。

## v2 - 關卡與碰撞

- 建立兩段關卡：星塵草原與星霧高塔。
- 加入地面缺口、高低平台、星磚、問號磚與終點門。
- 實作平台與磚塊的水平 / 垂直碰撞解析。

## v3 - 冒險元素

- 加入星幣、浮動分數、問號磚獎勵與 50 幣 1UP。
- 加入敵人巡邏、跳躍型敵人、踩踏擊倒、受傷、護盾與復活。
- 加入檢查點，啟動後失誤會從星旗附近復活。

## v4 - 道具與表現

- 加入護盾、跳靴、破磚火花三種道具。
- 加入粒子、跑步塵土、頂磚碎屑、背景雲層與多層山景。
- 加入響應式 HUD、任務面板與手機觸控操作。

## v5 - QA 後調整

- 用 Headless Chrome 驗證起始畫面、開始遊戲、跳躍、收集、Canvas 像素輸出與 HUD 更新。
- 第一輪測試發現第一隻敵人過早，容易在玩家熟悉操作前扣命，因此後移到第一個缺口之後。
- 降低雲層數量與透明度，讓前景平台、金幣與敵人更清楚。

## v6 - 流程驗證

- 驗證手機寬度無水平溢出，觸控按鈕正常顯示。
- 驗證抵達終點會進入關卡突破 Overlay。
- 驗證按下繼續後會正確載入 1-2，並把玩家放回新關卡起點。

## v7 - 尺寸、手感、動畫與音訊重做

- 舞台改為視窗高度約束的 16:9 顯示，避免 Canvas 高度與視窗不符。
- 降低重力並提高跑速、跳高與衝刺跳距，普通跳能撞到問號磚，衝刺跳能跨過常見缺口。
- 改善跳躍放開時的高度削減，不再因短按就完全跳不到磚塊。
- 加入落地壓縮、跑步擺腿、帽子與身體 bob、跳躍/落地/金幣/踩敵/道具/通關特效環。
- 加入 Web Audio 合成背景音樂與事件音效，開始遊戲後自動啟動。
- 另建立 `VERSION_300_UPGRADE_LOG.md`，把這輪需求拆成 v001-v300 的版本化迭代紀錄。

## v8 - 最終 QA 修正

- 補上 `Space` 鍵值相容，避免部分環境用 `Space` 而不是空白字元時無法跳躍。
- 手機版開始 Overlay 壓縮為短標題與按鈕，確保 16:9 舞台內按鈕完整可見。
- 驗證問號磚可被普通跳撞到，衝刺跳可跨過第一個常見缺口且不扣命。
